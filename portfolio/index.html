Business problem

High 30-day churn rate

Solution

Improve retention with additional tracking mode

Design process

Prepared task in Jira, added a project in Figma by a template

Made a discovery and documented a problem statement

  • There‚Äôs an option in apps to track in minutes (1 minute, 5 minutes, 10 minutes, 15 minutes, 30 minutes, 45 minutes, 60 minutes, 120 minutes, 240 minutes)
  • Time plays a significant role in finding a pet as a moving object, and even 1 minute delay is too much for this task. This was validated by several internal and external user interviews by product managers

Also documented an additional context

  • ‚ÄúLive tracking‚Äù mode requires exceptional indication as it should be limited by 15 minutes because of hardware limitations (battery drainage) and software limitation (high server load and package traffic limit)
  • ‚ÄúLive tracking‚Äù mode will be used as a core to define new subscription plans

Defined a design solution

Provide users with:

  • An option to activate "Live tracking" tracking mode
  • An indication of time left for an activation
  • An indication of number of feature activations left for the current day (because of the limitation)
  • An information on when number of feature activations resets (because of the limitation)

Analyzed how switching of tracking modes works now in iOS and Android apps

Started with an early concept: Decided where the switcher and indicator of Live tracking mode should be located

Decided what icons should be used

Defined states (a behavior) for the switch/indicator

Created a user flow with different scenarios (including corner/edge cases)

  • 5, 4, 3, 2, 1 activations left
  • None activations left
  • Timer starts
  • Less than 30 seconds of timer left
  • Timer ends

Refined the concept according to internal (product manager, developers) and external (loyal users) usability testings, feedback loops and discussions

Applied white-label styles and components

Simplified the element, made it more compact

Added subflow when there are no activations per day left

Added, tested and removed notifications when timer starts, ends or stops

Added a functionality to stop the timer

Added confirmation when users stop the timer

Added multiple devices behavior

  • Tracking mode affects all turned on devices, so badges of them all are changed to "Live tracking"
  • ‚ÄúUpdate location of all trackers‚Äù button now has ‚ÄúDisabled‚Äù state during ‚ÄúLive location‚Äù

And then:

  • Got developers feedback on refinemens
  • Created ready for development assets
  • Added acceptance criteria
  • Updated design library
  • Together with a product manager defined events for analytics
  • Supported developers during the implementation

Results

  • Most of monthly active users from beta testing group tried ‚ÄúLive tracking‚Äù functionality
  • Most of them were using ‚ÄúLive tracking‚Äù functionality at least once per two days (2-month statistics)
